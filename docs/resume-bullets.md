## 2026-07-04 — Phase 5 (5.1-5.3)

- **AI Agent Orchestration (Claude Agent SDK)** — Built and deployed a Claude Agent SDK-based SRE
  agent that authenticates through a federated MCP gateway with a team-scoped, non-admin token

  Stack: Claude Agent SDK, IBM ContextForge, MCP, Python, Server-Sent Events.

- **Agent-to-Agent (A2A) Protocol / Multi-Agent Delegation** — Implemented a two-agent system where
  a LangGraph coordinator delegates sub-tasks to a specialist agent entirely through a gateway's A2A
  integration (not a direct function call), with explicit checkpointed state and a first-class
  recoverable failure/retry path for delegation errors — verified end-to-end against a live
  production gateway.
  Stack: LangGraph, A2A protocol, IBM ContextForge, Azure Kubernetes Service, FastAPI.

- **Zero-Trust Agent RBAC Design** — Extended a gateway's "virtual servers as RBAC boundary" pattern
  from human teams to AI agents themselves, scoping a coordinator agent's callable surface to
  exactly one delegation tool so its own model could not bypass delegation and call downstream
  infrastructure tools directly — a capability-scoping technique applicable to any multi-agent
  system with a trust hierarchy.
  Stack: ContextForge RBAC, virtual servers, JWT token scoping, REST API-driven config.

- **CI/CD with Zero-Secret Cloud Auth (GitHub Actions + Azure OIDC)** — Designed and stood up a
  GitHub Actions pipeline with a security-first split: an unguarded CI job authenticates via a
  Reader-only OIDC federated identity, while the actual deploy job runs inside a required-reviewer
  GitHub Environment using a second, more privileged OIDC identity whose federated-credential
  subject is scoped to that environment alone — no long-lived cloud secrets stored anywhere.
  Stack: GitHub Actions, Azure AD Workload Identity Federation (OIDC), Bicep, Helm.

- **Production Network Security Debugging (Kubernetes NetworkPolicy)** — Diagnosed a fully-hung
  service-to-service call in a zero-trust AKS cluster by isolating the failure to one workload's own
  egress rules via direct in-pod connectivity testing (`kubectl exec` + `curl`) against a working
  `kubectl port-forward` baseline, then closed the gap with a least-privilege NetworkPolicy rule
  scoped to the destination pod's real container port rather than the Service's exposed port.
  Stack: Kubernetes NetworkPolicy, AKS, kubectl, Calico.

- **Infrastructure-as-Code Drift Prevention (Bicep what-if)** — Caught a node-pool scale-down
  regression before it reached production by running `az deployment sub what-if` ahead of a routine
  Bicep deployment, tracing the root cause to a stale parameter the prior incident fix had missed —
  reinforcing what-if as a standing pre-deploy habit rather than a one-time fix.
  Stack: Bicep, Azure CLI, AKS, Infrastructure as Code.
